Giới thiệu

Mục tiêu và phạm vi của tài liệu

Tài liệu này định nghĩa Web Tracking Methodology cho toàn bộ hệ thống web của MoMo — bao gồm 100+ landing pages, 20+ use cases, và hàng chục product teams. Mục tiêu là đảm bảo mọi use case khi triển khai tracking đều tuân theo một chuẩn thống nhất, tránh data fragmentation và dễ maintain về lâu dài.

🎯

Ai dùng tài liệu này?

Dev, PO, SEO/GEO Lead — bất kỳ ai tham gia vào quá trình thiết kế hoặc triển khai tracking cho một use case mới.

📐

Phạm vi

Web tracking only: GTM (client-side), GA4 (data collection), Looker Studio (reporting). Không bao gồm App tracking (Appsflyer/Firebase).

🔄

02

Nguyên tắc cốt lõi

5 nguyên tắc bất biến khi thiết kế tracking

1️⃣

dataLayer-first

Tất cả events phải được fire thông qua dataLayer.push() từ Dev. Không dùng GTM Click/Visibility trigger tự detect — dễ vỡ khi thay đổi UI.

2️⃣

Payload pattern

Tất cả params gom vào object payload: {} — nhất quán, dễ debug, và tránh naming conflict giữa các use cases.

3️⃣

Shared dimensions

Ưu tiên dùng shared params (use_case, payment_method...) thay vì tạo riêng cho từng use case. GA4 Free chỉ có 50 event-scoped dimensions.

4️⃣

Naming consistency

Event name theo pattern {usecase}_{action}. DLV theo pattern DLV - {project} - {key}. Không deviation.

5️⃣

Verify trước Publish

Bắt buộc test qua GTM Preview + GA4 DebugView trước khi Submit/Publish. Không Publish production mà chưa verify event fired.

Tiếp cận Use Case mới

Checklist và quyết định khi onboard use case mới

Phân loại Page Type trước khi thiết kế tracking

Advanced MiniWeb

Multi-step flow

Có simulator, API call, nhiều màn hình. VD: Cinema, BHYT, OTA Bus, Vay nhanh. Cần track đầy đủ funnel từ view → submit → payment → result.

Basic MiniWeb

Single page CTA

Landing page đơn giản, 1 CTA chính dẫn vào App. VD: Chuyển tiền, Billpay, Tiết kiệm. Chỉ cần track page_view + click_to_app.

Campaign MiniWeb

Thời vụ

Campaign có thời hạn. VD: Lắc Xì, Mega, KBBX. Track engagement + CTA click. Cần tag cleanup sau campaign.

GTM Setup

Hướng dẫn tạo Variables, Triggers, Tags theo chuẩn

Bước 1 — Tạo Data Layer Variables (DLV)

GTM / DLV Config
// Naming pattern: DLV - {project} - {key}
// VD: DLV - bhyt - lookupType

Variable Type      : Data Layer Variable
DLV Key            : payload.lookupType   ← nested nếu dùng payload{}
Default Value      : (not set)          ← LUÔN set default
Data Layer Version : 2
⚠️ Quan trọng Luôn bật "Set Default Value" và điền (not set) — tránh GA4 nhận undefined khi Dev quên push param.

Bước 2 — Tạo Custom Event Trigger

GTM / Trigger
Trigger Name : 1_bhyt_form_view    ← {number}_{event_name}
Trigger Type : Custom Event
Event Name   : bhyt_form_view       ← match EXACTLY với dataLayer event
Match Type   : Equals               ← không dùng regex trừ catch-all

Bước 3 — Tạo GA4 Event Tag

GTM / Tag
Tag Name          : [BHYT] 1-bhyt_form_view
Tag Type          : Google Analytics: GA4 Event
Folder            : [S] - BHYT
Configuration Tag : ← GA4 Config tag hiện có
Event Name        : bhyt_form_view

Event Parameters:
  use_case      → {{DLV - bhyt - use_case}}
  lookupType    → {{DLV - bhyt - lookupType}}
  insuranceStatus → {{DLV - bhyt - insuranceStatus}}

Firing Trigger    : 1_bhyt_form_view

Folder structure trong GTM

PrefixÝ nghĩaVí dụ
[S]Service / Product funnel tracking[S] - BHYT, [S] - Cinema, [S] - OTA BUS
[P]Product / Ads widget[P] - HELLOMOMO, [P] - WebAds
[C]Campaign (thời vụ)[C] - LACXI25, [C] - MEGA25

dataLayer.push() spec cho Dev

JavaScript / dataLayer
window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
  event: "bhyt_form_submit",          // event name — match GTM trigger
  payload: {
    use_case:        "bhyt",         // REQUIRED — shared dimension
    lookupType:      "cccd",         // medical_number | personal_id
    insuranceStatus: "active",       // active | expired | expiring
    statusCode:      "46",           // error code nếu có
    paymentMethod:   "momo_wallet",  // shared param
  }
});

Quy trình register Custom Dimension mới

  • GA4 → Admin → Custom Definitions → Create custom dimension
  • Điền Dimension name (display label), Scope = Event, Event parameter = exact key name
  • Save → chờ 24-48h để data backfill
  • Looker Studio → Resource → Manage data sources → REFRESH FIELDS
  • Data cũ trước khi register sẽ là (not set) — không thể backfill

Looker Studio

Calculated Fields, phân loại URL, và các pattern reporting

Phân loại URL — 2 Calculated Fields bắt buộc

FieldReturn ValuesDùng để
CONTENT_TYPE Advanced MiniWeb, Basic MiniWeb, Campaign MiniWeb, Blog, News, FAQ, Guide, Merchant MiniWeb, Others Phân loại loại trang — so sánh performance theo page type
PRODUCT MDS-Cinema, FS-BHYT, UTI-Telco, DLS-Digital Entertainment... Phân loại team/product — báo cáo theo từng PO
Rule quan trọng — REGEXP_MATCH vs REGEXP_CONTAINS REGEXP_MATCH: match toàn bộ string — cần .* ở đầu/cuối pattern.
REGEXP_CONTAINS: tìm substring — không cần .*.
Sai function → data null toàn bộ.

Pattern tính metrics cross-event

Looker Studio / Calculated Field
// ❌ SAI — row-level, luôn ra 0% hoặc 100%
CASE WHEN Event Name = "popup_click" THEN 1 ELSE 0 END
/ CASE WHEN Event Name = "popup_view" THEN 1 ELSE 0 END

// ✅ ĐÚNG — aggregate trước khi chia
SUM(Popup Clicks) / SUM(Popup Views)

// Chart filter khi dùng pattern này:
Event Name REGEXP_MATCH webads_popup.*   ← bắt cả view + click

Hyperlink field cho Page URL

Looker Studio / Calculated Field
HYPERLINK(
  CONCAT("https://www.momo.vn", Page path and screen class),
  "Link"
)

Naming Convention

Quy tắc đặt tên bắt buộc cho toàn bộ hệ thống

Thứ gìPatternVí dụ
Event Name {usecase}_{action} bhyt_payment_click, cinema_booking_view
GTM Tag [FOLDER] {N}-{event_name} [BHYT] 1-bhyt_form_view
GTM Trigger {N}_{event_name} 1_bhyt_form_view
DLV DLV - {project} - {key} DLV - bhyt - lookupType
use_case value lowercase, no space bhyt, cinema, travelsim
GA4 Custom Dimension Title Case — mô tả rõ nghĩa Ad Campaign, Ad Project
Event param key snake_case payment_method, insurance_status
⚠️ Tuyệt đối không Dùng camelCase cho event param keys (paymentMethod → sai), dùng dash trong event name (bhyt-form-view → sai), đặt tên param quá generic không có context (status, type).

Custom Link Type — Variable JS

Variable này tự động phân loại loại link khi user click CTA. Hiện support 3 OneLink domains của MoMo:

GTM / Custom JavaScript Variable
function() {
  var clickURL = String({{Click URL}} || '');
  var pageURL  = String({{Page URL}} || '');

  if (pageURL.indexOf('popup-qrcode') > -1) return 'universal_qrcode';
  if (clickURL.indexOf('onelink') > -1)    return 'onelink';
  if (clickURL.indexOf('momoapp') > -1)    return 'onelink';
  if (clickURL.indexOf('momo://?refId=') > -1) return 'refid_link';
  if (clickURL && !{{Outbound Links}})       return 'internal_link';
  if (clickURL && {{Outbound Links}})        return 'outbound_link';
}

Looker Studio Formulas

Các Calculated Fields chuẩn — copy & paste

CONTENT_TYPE — Phân loại loại trang

Looker Studio / Calculated Field — GSC Data Source
CASE
WHEN REGEXP_CONTAINS(Landing page,'momo.vn/$')             THEN "Advanced MiniWeb"
WHEN REGEXP_MATCH(Landing page,'.*/thanh-toan-momo-.*')       THEN "Merchant MiniWeb"
WHEN REGEXP_MATCH(Landing page,'.*/doi-tac.*')               THEN "Merchant MiniWeb"
WHEN REGEXP_MATCH(Landing page,'.*/hoi-dap.*')               THEN "FAQ"
WHEN REGEXP_MATCH(Landing page,'.*/blog.*')                  THEN "Blog"
WHEN REGEXP_CONTAINS(Landing page,'/tin-tuc')                THEN "News"
WHEN REGEXP_MATCH(Landing page,'.*/huong-dan.*')             THEN "Guide"
WHEN REGEXP_MATCH(Landing page,'.*/tim-kiem.*')              THEN "Search"
WHEN REGEXP_CONTAINS(Landing page,'/cinema|/vay-nhanh|/ve-xe|/bao-hiem-o-to|/bao-hiem-xe-may|/bao-hiem-y-te|/du-lich-di-lai') THEN "Advanced MiniWeb"
WHEN REGEXP_CONTAINS(Landing page,'/lac-xi|/bonbon|/super-mega|/tro-thu-tai-chinh') THEN "Campaign MiniWeb"
WHEN REGEXP_CONTAINS(Landing page,'/chuyen-tien|/tiet-kiem-online|/thanh-toan-hoa-don|/nap-tien-dien-thoai') THEN "Basic MiniWeb"
ELSE "Others"
END
⚠️ Thứ tự WHEN CASE/WHEN xử lý từ trên xuống — rule đặc biệt hơn phải đứng trước rule chung hơn. /cinema/blog/... sẽ match /blog hoặc /cinema tùy rule nào đứng trước.

WebAds Popup Metrics

// Popup Click — Aggregation: SUM

QA Checklist

Các bước verify trước khi Publish GTM

GTM Preview

  • Mở GTM Preview → vào trang cần test → trigger action tương ứng
  • Summary panel: event name hiện đúng trong list
  • Tag status = Fired (không phải Not Fired)
  • Tab Variables: DLV trả về giá trị đúng, không phải undefined
  • Tag Output: tất cả event parameters có giá trị, không có undefined

GA4 DebugView

  • GA4 → Configure → DebugView → trigger event
  • Event xuất hiện trong timeline với đúng event name
  • Click vào event → xem parameters list → tất cả params có giá trị
  • Không có parameter nào trả về (not set) ngoài dự kiến

Test bằng Console (không cần Dev deploy)

Browser Console / Simulate event
// Paste vào Console để test — không ảnh hưởng production data
window.dataLayer.push({
  event: "bhyt_form_submit",

    use_case:        "bhyt",

    insuranceStatus: "active",
  }

Sau khi Publish

  • GA4 Realtime → Events → confirm event xuất hiện với đúng tên
  • Chờ 24-48h → GA4 Explore → verify custom dimension có data
  • Looker Studio → Refresh Fields → confirm field mới xuất hiện
  • Update GTM Inventory CSV với Tag ID thật (thay "NEW")
